# Remix This Effect

Use this skill to preserve an animation mechanic while changing its purpose and expression.

## Process

1. Understand the source effect:
   - visual mechanic
   - interaction input
   - timing/easing style
   - required DOM/CSS/assets
   - reusable variables

2. Understand the target:
   - product category
   - page section
   - content model
   - brand tone
   - device constraints

3. Separate mechanic from skin:
   - mechanic: timeline, input mapping, transforms, masks, camera movement
   - skin: copy, assets, colors, layout, type, density, pacing

4. Redesign the content model:
   - project cards
   - product variants
   - case-study steps
   - feature states
   - gallery items
   - menu destinations

5. Implement the remix:
   - rename domain objects
   - replace hardcoded media with data
   - expose useful parameters
   - keep animation code local to the component/page
   - include mobile and reduced-motion fallbacks

## Output

Provide:

- remix concept
- source mechanic retained
- changes made
- new content/data shape
- implementation steps
- risks and fallbacks

## Taste Rules

- Do not clone the original visual identity.
- Keep the motion reason tied to the product.
- Remove any effect that competes with reading or buying.
- Prefer a cleaner remix over a louder one.
